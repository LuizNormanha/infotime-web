PROMPT IDEAL:

Quero criar um CRUD de convênio seguindo o padrão Infolab.

Antes de gerar código:
1) Confirme comigo se preciso de listagem + formulário ou só uma das partes.
2) O formulário será só leitura em algum caso?
3) Cadastro em uma tela só ou em etapas?
4) Aplicar regra de campos obrigatorios para quais campos alem dos já exigidos no Schemas?

Depois das minhas respostas, chame a tool MCP infolab.crud_briefing com um JSON refletindo o que combinamos (dominio: convenio, comListagem, comFormulario, etc.) e use o briefing para implementar listagem e formulário em web + Nest.
